# AI Disclosure

Some parts of the historical codebase used AI Generated code, but those elements are no longer present in the codebase, having been rewritten entirely. (Specifically, the first version of the OverlayService)
No part of this library is vibe-coded or contains generative AI generated assets.
AI Tools were used for research and learning purposes but no code was directly generated by AI.
In a modern programming environment, it is difficult to completely avoid all AI influence (e.g., Google AI responses to searches, IDE autocomplete and auto format).
Archipelago.Core relies on framework from other projects, as well as multiple community libraries, over which I have little or no influence. To the best of my understanding, generative AI usage in these is minimal but non-zero.
There are no plans to add generative AI generated code in the future.

The documentation for this project (README and files under `docs/`) was generated with the assistance of an AI language model (Claude by Anthropic).
